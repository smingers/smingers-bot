## 7. Recommendations

### Research Improvements

1. **Prioritize actual data retrieval**: When the agent identifies that historical Google Trends data is critical but not retrieved, the pipeline should flag this as a research failure requiring remediation before forecasting proceeds
2. **Cross-validate news items**: Stories mentioned in agent reports (like Feb 2 news items) should be confirmed in AskNews or other current search results before being used as forecast drivers
3. **Include day-of-week analysis**: For Google Trends questions, research should explicitly pull weekday/weekend patterns for the specific term

### Prompt/Pipeline Improvements

1. **Standardize structural factor treatment**: Add explicit guidance on whether/how to weight day-of-week effects for Google Trends questions
2. **Flag data gaps more prominently**: When critical reference class data is unavailable, the prompt should explicitly instruct forecasters how to handle this uncertainty
3. **Add consistency checks**: Flag when forecasters' probability estimates diverge by >30pp on the same outcome
