## 7. Recommendations

### Research Improvements
1. **Prioritize retrieving the actual Metaculus community prediction value.** For meta-questions about community predictions, the single most important data point is the current level of that prediction. The pipeline should attempt authenticated API access or page scraping of the Metaculus question before any other research.
2. **Add prediction market query targets.** For questions about crowd forecasts, Polymarket and Kalshi prices are directly comparable anchors. The query generation should include explicit prediction market searches.
3. **Add Metaculus-specific behavioral queries.** Queries like "how do Metaculus community predictions drift over 1-week periods" or "Metaculus binary prediction volatility patterns" would have been directly relevant.

### Prompt/Pipeline Improvements
1. **Increase max_output_tokens for GPT-5.2.** The 4000-token limit caused S1-3 to be truncated, a critical failure. GPT-5.2 should have at least 5000-6000 tokens, or the limit should be model-specific (as it appears to be for Sonnet 4.5 in the config).
2. **Handle missing S1 outputs gracefully.** When S1-3 fails, the pipeline should either (a) retry with a higher token limit, (b) route S2-4 to a different S1 output, or (c) flag that S2-4 will lack cross-pollinated input. Currently it appears S2-4 received no valid outside view and had to construct its own base rate, leading to the outlier.
3. **Explicitly surface key provided data points in prompts.** The question description contained "The current community prediction as of 2026-02-01: 15.00%" -- this should be highlighted or extracted and injected into the outside view prompt separately from the bulk research context, so forecasters cannot overlook it.

### Model-Specific Feedback
1. **Sonnet 4.5 (Forecasters 1 & 2):** Solid qualitative reasoning but could benefit from more quantitative frameworks. S2-1 under-updated from its outside view (+1pp despite substantial new evidence).
