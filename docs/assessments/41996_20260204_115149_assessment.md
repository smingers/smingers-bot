## 7. Recommendations

### Research Improvements

1. **Add a financial data API source:** For stock price questions, the research pipeline should include a dedicated financial data source (e.g., Yahoo Finance API, Alpha Vantage) that can retrieve historical return distributions for the specific ticker over comparable time horizons. This would provide verified base rate data rather than forcing forecasters to rely on training data or fabricate statistics.

2. **Filter AskNews by ticker relevance:** Implement post-retrieval filtering to remove articles that mention unrelated tickers with similar names (e.g., RCF.NS, RCOM.NS when searching for NYSE:RCL). This would reduce context window waste and improve signal-to-noise ratio.

3. **Add intraday price data for same-day forecasts:** When the forecast is generated on the same day as the baseline close (Feb 4), include real-time price data or at minimum the previous close as a more reliable anchor than pre-market Stockpulse quotes.

### Prompt/Pipeline Improvements

1. **Ensure Feb 4 close is explicitly provided to all forecasters:** The critical baseline price ($326.29) was only available in current search results (AskNews Stockpulse article), not in the outside view prompt. The inside view prompt should explicitly state the confirmed baseline close price to prevent forecasters from using stale estimates.

2. **Consider weighting recent price data more heavily for stock price questions:** For short-term stock price direction questions, the inside view update is doing the most important analytical work. Consider whether the pipeline should provide more structure for how same-day price information should be integrated.
