## 7. Recommendations

### Research Improvements

1. **Use authenticated API for Metaculus meta-questions**: When the question involves predicting community prediction movement, the bot should query the Metaculus API (using its existing token) to retrieve the current CP value and recent trajectory for the target question. This is the single most valuable data source for this question type and was entirely missed.
2. **Add economic data calendar queries**: For short-horizon meta-questions about economic indicators, include a query specifically targeting upcoming scheduled data releases (e.g., "US economic data releases February 2026 calendar") to help forecasters assess whether catalyst events fall within the prediction window.
3. **Filter irrelevant AskNews results**: The AskNews query returned numerous irrelevant international articles (Italian GDP, Romanian GDP, Brazilian politics). The query should be more specific to US economic data or the AskNews results should be filtered for relevance before being passed to forecasters.

### Prompt/Pipeline Improvements

1. **Add guidance for meta-questions**: The prompts should include specific guidance for "community prediction rises" questions, emphasizing that the key question is about CP drift dynamics rather than the underlying event probability. Forecasters should be instructed to distinguish between "what should the probability be?" and "will the community prediction move upward?"
2. **Consider spread-aware aggregation**: When the ensemble spread exceeds a threshold (e.g., 30pp for binary), consider using median instead of mean, or applying a trimmed mean to reduce the influence of outlier predictions.
3. **Add status quo framing for threshold questions**: For questions where the current value is exactly at or near the resolution threshold, the prompt should explicitly note that "no change" resolves to a specific outcome (in this case, No), as this is a critical framing that some forecasters missed.

### Model-Specific Feedback

- **Sonnet 4.5 (S2-1, S2-2)**: Produced the widest divergence in the ensemble (25% and 62%). S2-1 was too aggressive in its downward adjustment (-10pp), while S2-2 was too aggressive upward (+7pp). Both would benefit from more conservative update magnitudes and explicit cross-checking against the base rate.
- **GPT-5.2 (S2-3)**: Strongest individual performance. The status quo asymmetry insight was the most analytically valuable contribution. No model-specific improvements needed.
- **o3 (S2-4, S2-5)**: Both produced reasonable middle-ground predictions (51%, 37%). However, both cited unverifiable historical statistics about Metaculus CP drift patterns. The pipeline should either validate such claims or instruct models to note when they are citing recalled knowledge vs. sourced evidence.
