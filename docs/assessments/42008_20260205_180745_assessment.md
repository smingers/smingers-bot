## 7. Recommendations

### Research Improvements

1. **Integrate a financial data API for stock price questions.** The agentic search spent 7 steps trying to download HAL price data from Stooq, Alpha Vantage, and Yahoo Finance via Google search -- and failed. For "close_price_rises" format questions, the pipeline should directly call a financial data API (e.g., Yahoo Finance yfinance Python library, or Alpha Vantage API) to retrieve historical daily prices and compute the empirical return distribution. This is the single highest-impact improvement for stock price questions.

2. **Improve ticker disambiguation for AskNews.** AskNews returned mostly articles about Hindustan Aeronautics Limited (Indian exchange "HAL"), HAL Trust (Amsterdam), and Halberd Corp. The query should explicitly include "Halliburton NYSE" or the full company name to reduce false matches. Consider post-processing AskNews results to filter by exchange or country.

3. **Filter irrelevant scraped articles before including in context.** The Eversource and Skyworks articles consumed context window space without adding value. A simple relevance check (does the article mention the target company?) would have filtered these out.

### Prompt/Pipeline Improvements

1. **Consider capping additive adjustment magnitude.** S1-1's +8pp for momentum is aggressive. The prompts could suggest a maximum per-factor adjustment (e.g., +/-5pp) to prevent any single qualitative factor from dominating the base rate calculation.

2. **Encourage forecasters to explicitly state when empirical claims come from training data vs. research context.** S1-4 and S1-5 cited plausible but unverifiable statistics about HAL's return distribution. The prompt could ask forecasters to tag claims as "from research context" vs. "from general knowledge" to improve transparency.
