## 7. Recommendations

### Research Improvements

1. **Add early termination/pivot logic to agentic search.** When the agent determines (after 2-3 steps) that authenticated access is required, it should pivot to alternative queries rather than spending 4-5 more steps on the same dead end. For this question, the agent could have searched for "Metaculus Greenland question community prediction screenshot" or "Metaculus 41502 current probability" to find blog posts or social media discussions that mention the current CP.

2. **Use the bot's own Metaculus API token for prediction timeseries.** The bot has `METACULUS_TOKEN` in its environment for submitting predictions. This same token could be used in the agentic search to directly query the Metaculus API for the target question's CP timeseries, which would have been the most valuable piece of data for this meta-prediction question.

3. **For meta-prediction questions ("will CP rise/fall"), prioritize accessing the current CP value.** The bot should recognize that the single most important data point is the current community prediction, and dedicate research resources to obtaining it. A direct API call to `/api2/questions/41502` with the bot's token would trivially resolve this gap.

### Prompt/Pipeline Improvements

1. **Consider providing the current CP value in the prompt when available.** The question JSON already contained `"last_cp": 0.1` in the metadata field. This was visible in the prompt, but a more explicit presentation (e.g., "The current community prediction as of question launch is 10.00%") would help forecasters anchor to the known value more clearly.

2. **Add guidance for meta-prediction questions.** The prompt could include a note for "metaculus_binary_cp_rises" format questions: "This is a meta-prediction question about Metaculus crowd behavior, not about the underlying event. Focus your analysis on prediction dynamics (volatility, drift, stickiness) rather than geopolitical analysis."

3. **Surface cross-pollinated base rate more explicitly in inside view prompt.** To reduce the base rate confusion observed in S2-4 and S2-5, the inside view prompt could more prominently state: "Your outside view base rate is X%. This was produced by [model name]." This would reduce instances where forecasters reference internally derived base rates instead of the cross-pollinated input.
