## Research Quality and Base Rate (Emphasis)

### Was a good base rate for Apple and Nvidia stock price history produced?

**No.** A proper *empirical* base rate for the *biweekly return spread* (NVDA_return − AAPL_return) was never produced.

- **Programmatic stock return module:** Not used. The pipeline’s `stock_return` module (`compute_stock_return_distribution`) only runs for questions whose description contains `"close_price_rises"` (single-ticker close price). This question uses `"metac_reveal_and_close_in_period"` and asks for the *difference* of two tickers’ returns, so the programmatic stock-return distribution was never computed and no `search_stock_return_distribution.json` artifact exists for this run.
- **yFinance (planned and executed):** Queries for NVDA and AAPL both succeeded. The historical context contained:
  - **Per-ticker:** current price, previous close, 1yr/5yr price stats (mean, min, max, std dev), last 10 trading days of daily prices, options-implied volatility, analyst targets, recent return changes (1w, 1m, 3m, 6m).
  - **What was not present:** Any long-horizon series of *biweekly* returns or of the *spread* (NVDA − AAPL) over 10-trading-day windows. So yFinance supplied inputs useful for volatility and recent levels but not a distribution of the resolution metric.
- **Agent report (historical base-rate query):** The Agent was asked to analyze past 10-trading-day NVDA−AAPL return differentials 2016–2025 and compute mean, stdev, percentiles. It explicitly stated that “the raw day-by-day series needed to compute rolling 10-day returns were not returned in the snippets above – only annual summaries and a handful of 2026 daily prints. Consequently, the exact numerical distribution of 10-day spreads cannot be calculated from the material provided.” It then gave **indicative** moments (mean ~+2–3 pp, stdev ~8–10 pp, 5th ~−15 pp, 95th ~+20 pp) and qualitative tail examples, labeling them “placeholders” and “reasonable expectation” rather than computed statistics.

So the only “base rate” that reached forecasters was this **indicative, non-computed** summary from the Agent, plus yFinance’s per-ticker stats and recent prices. No one in the pipeline had access to a full empirical distribution of historical biweekly NVDA−AAPL spreads.

### How / when / where was the base rate propagated?

- **Where:** The Agent report and yFinance blocks were included in the **historical context** passed to all five outside-view (Step 1) instances via `ensemble/outside_view_prompt.md` (and the underlying research assembly). - **When:** During the research “execute” and “assemble” phases; that combined historical context was then used in the single outside-view prompt for all five forecasters.

## 8. Recommendations

### Research Improvements

- Extend the pipeline to support **two-ticker return-spread** questions (e.g. NVDA−AAPL over N trading days): compute rolling spread from yFinance (or similar) and inject a programmatic “spread distribution” context, analogous to single-ticker `stock_return` for close-price questions.
- Add fallback or logging when question-embedded URLs (especially resolution-criteria data sources) return 4xx/5xx so that missing scrape data is explicit in artifacts and assessments.

### Prompt/Pipeline Improvements

- For numeric questions that resolve as a difference of two series (e.g. return spread), consider prompting the planner to request or the pipeline to compute an empirical distribution of that difference when tickers and window are identifiable.
